## 背景

42Tokyoの課題は、提出先としてintra内に`vogsphere`というGitリポジトリが用意されている。ただし素のGitリポジトリのみの提供であるため、そのままでは使いにくい点がある。

- UIがなく、コードをブラウザで確認できない
- CIを走らせられない

そこでGitHubをメインの管理場所として使いつつ、提出時だけvogsphereにpushする運用にすることにした。

また、42ではコーディング規約`Norm`に準拠したコードを書く必要がある。これは、機械採点時にチェックが走るほか、配布されているチェックツール`Norminette`を利用することで、自分でもチェックすることができる。GitHubで管理することにより、CIでNorminetteを走らせることができるのも大きなメリットになる。

## GitHubでの管理

以下の構成をテンプレートとして用意し、課題ごとにテンプレートからリポジトリを作成することにした。

```bash
.
├── Makefile
├── srcs/        # ソースファイル
├── includes/    # ヘッダーファイル
└── .github/
    └── workflows/
        └── norminette.yml
```

MakefileはSRCS変数にファイル名を追加するだけでビルドできるシンプルな構成にしている。

## リモートの設定と提出

GitHubリポジトリを`origin`として作成した後、intraで払い出されたvogsphereのURLを`vogsphere`という名前でリモートに追加する。

```bash
git remote add vogsphere git@vogsphere.42tokyo.jp:intra-uuid-xxx/...
```

`git remote -v`で両方が登録されていることを確認できる。

```bash
origin      git@github.com:yourname/repo.git (fetch)
origin      git@github.com:yourname/repo.git (push)
vogsphere   git@vogsphere.42tokyo.jp:intra-uuid-xxx/... (fetch)
vogsphere   git@vogsphere.42tokyo.jp:intra-uuid-xxx/... (push)
```

普段はoriginにpushしてCIを回し、提出タイミングだけvogsphereにpushする運用となる。

```bash
git push vogsphere main
```

## GitHub ActionsでNorminetteを自動実行

`main`ブランチへのpushのタイミングでNorminetteが自動実行されるようにした。

```yaml
- name: Run norminette
  id: norminette
  run: |
    set +e
    OUTPUT=$(norminette 2>&1)
    EXIT_CODE=$?
    echo ""
    {
      echo "output<<EOF"
      echo "$OUTPUT"
      echo "EOF"
    } >> $GITHUB_OUTPUT
    exit $EXIT_CODE
```

実行結果を`GITHUB_OUTPUT`に保存しておくことで、後続のステップから参照できる。

## Discord通知

CIの結果をDiscordに通知するようにした。`jq`でJSONを組み立てることでコミットメッセージ内の特殊文字によるJSONパースエラーを防いでいる。

通知には以下の情報を含めている。

- 成功 / エラーをタイトルと色で視覚的に区別
- リポジトリ名・ブランチ名
- コミットメッセージ（全文）
- 実行者
- GitHubのコミットURL
- Norminetteの実行結果（コードブロック）

Discord Webhookの設定はオプションにしており、Secretを登録していない場合は通知ステップが自動スキップされる。テンプレートリポジトリとして配布しやすいようにした。
